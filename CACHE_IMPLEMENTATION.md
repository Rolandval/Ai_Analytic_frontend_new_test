# 📦 Система кешування даних для AI Product Filler

## Огляд

Реалізовано систему попереднього кешування даних для товарів та категорій з метою покращення UX та зменшення часу очікування користувача при завантаженні сторінки.

## Архітектура

### Компоненти системи

1. **`dataCache.ts`** - Утиліта для роботи з IndexedDB
2. **`useCachedData.ts`** - React хук для керування кешованими даними (опціонально)
3. **`Generation.tsx`** - Оновлений компонент з інтеграцією кешування

## Принцип роботи

### 1. Попереднє кешування

При першому завантаженні даних з API:
- Зберігаються **перші 50 записів** у IndexedDB
- Кеш зберігається окремо для кожної мови (`ua`, `en`, `ru`)
- Термін дії кешу: **24 години**

### 2. Миттєве відображення з кешу

При наступних відвідуваннях сторінки:
```
1. Перевірка наявності кешу для поточної мови
2. Якщо кеш знайдено:
   ✅ Миттєво показати дані з кешу
   ✅ Приховати лоадер
   ✅ Показати індикатор "📦 Кеш"
3. Якщо кешу немає:
   ⏳ Показати лоадер
   ⏳ Завантажити дані з API
```

### 3. Фонове оновлення

Після показу кешованих даних:
```
1. Запускається фоновий запит до API (через 100ms)
2. Користувач продовжує працювати з кешованими даними
3. Після отримання свіжих даних:
   ✅ Плавне оновлення таблиці
   ✅ Оновлення кешу
   ✅ Приховування індикатора "📦 Кеш"
```

## Технічні деталі

### IndexedDB структура

```typescript
База даних: AiAnalyticCache

Сховища:

- categories // Категорії

Формат запису:
{
  data: T[],           // Масив даних (до 50 елементів)

  lang: string         // Мова даних
}
```

### Валідація кешу

Кеш вважається валідним якщо:
- ✅ Запис існує в IndexedDB
- ✅ Вік кешу < 24 години
- ✅ Мова кешу співпадає з поточною мовою

### Обробка помилок

Система розроблена з принципом **graceful degradation**:
- Помилки кешування не впливають на основний функціонал
- При помилці IndexedDB - звичайне завантаження з API
- Всі помилки логуються в консоль для діагностики

## Використання

### Автоматичне кешування

Кешування працює автоматично:
```typescript
// При завантаженні товарів
const fetchData = async (showLoader: boolean = true) => {
  // 1. Спроба завантажити з кешу
  if (showLoader) {
    const cachedData = await dataCache.getCachedProducts(selectedLang);
    if (cachedData) {
      // Показати кеш миттєво
      setDescriptions(cachedData);
      setIsDataFromCache(true);
      // Запустити фонове оновлення
      setTimeout(() => fetchData(false), 100);
      return;
    }
  }
  
  // 2. Завантаження з API
  const response = await fetchContentDescriptions(request);
  
  // 3. Збереження у кеш
  await dataCache.cacheProducts(response.items, selectedLang);
  setIsDataFromCache(false);
}
```

### Ручне очищення кешу

```typescript
// Очистити весь кеш
await dataCache.clearAll();

// Очистити тільки товари
await dataCache.clearCache('products');

// Очистити тільки категорії
await dataCache.clearCache('categories');
```

## Майбутні покращення

### Можливі оптимізації:
1. **Service Worker** - кешування на рівні мережі
2. **Compression** - стиснення даних перед збереженням
3. **Selective caching** - кешування на основі частоти використання
4. **Cache warming** - попереднє завантаження при idle
5. **Progressive loading** - поступове завантаження даних

### Додаткові функції:
1. **Налаштування** - користувач може вимкнути кеш
2. **Статистика** - показ розміру кешу та економії часу
3. **Sync** - синхронізація між вкладками
4. **Offline mode** - повна робота офлайн з кешем

## Тестування

### Перевірка роботи кешу:

1. **Перший візит**:
   - Відкрити DevTools → Network
   - Завантажити сторінку
   - Перевірити запит до API
   - Перевірити IndexedDB (Application → IndexedDB → AiAnalyticCache)

2. **Повторний візит**:
   - Оновити сторінку (F5)
   - Дані мають з'явитися миттєво
   - Індикатор "📦 Кеш" має бути видимим
   - Через ~100ms запуститься фоновий запит
   - Після завершення індикатор зникне

3. **Зміна мови**:
   - Змінити мову (ua → en)
   - Перший раз - завантаження з API
   - Повторно - завантаження з кешу

### Очищення кешу для тестування:

```javascript
// В консолі браузера:
indexedDB.deleteDatabase('AiAnalyticCache');
```

## Troubleshooting

### Кеш не працює?

1. **Перевірити IndexedDB**:
   - DevTools → Application → IndexedDB
   - Має бути база `AiAnalyticCache`

2. **Перевірити консоль**:
   - Шукати помилки з префіксом `[DataCache]`
   - Перевірити чи є логи про cache hit/miss

3. **Перевірити мову**:
   - Кеш окремий для кожної мови
   - При зміні мови завантажується новий кеш

### Дані не оновлюються?

1. **Фонове оновлення працює**:
   - Після показу кешу запускається фоновий запит
   - Перевірити Network tab через ~100ms після завантаження

2. **Примусове оновлення**:
   - Натиснути кнопку "Оновити"
   - Це завантажить свіжі дані з API
